- [苦しんで覚える C 言語](https://9cguide.appspot.com/index.html)

## 感想

- 本について
  - どんな言語でも他に学んだ経験があれば、全く苦しくはない。
  - C 言語はやっぱり直感的じゃない動作が沢山ある、歴史が長くコンピュータに少し近い言語なんだなと、読み始めてからずっと感じ続けてる。
  - 現行のバージョンから見ると、少し解説が古いところがあるようだったけど、基本的な C 言語の書き方や動作をおさえるのにちょうどいい一冊だ。
  - もう少し詳細な説明がほしいと思う人もいるかも知れないが、初めて C 言語を学ぶ人にとって、過不足なく飽きない長さで説明されている。
  - この本は、この説明でも分かるんだというほどに簡潔な説明で C 言語の直感的な理解を与えてくれる。
- 自分が学んできた言語と比較して、違い（あるいはそこから滲み出る歴史）を感じる点。
  - 初期化
  - 変数のスコープ
  - 文字列の扱い方（より高級な言語を先に学んできた自分にとってはかなり扱いにくい）
- C という言語について
  - PHP や Python や Go のような自分が扱ってきた高級言語を使うより遥かに、コンピュータの内部的な挙動を意識する機会、その恐怖が多い言語だな。
  - 「第15章 ポインタ変数の仕組み」の「&つけが必要な変数の正体」で C 言語の関数の引数についての認識が変わった。
  - C 言語では関数の引数はあくまでも数値であり '&' 演算子は、数値しか受け取れない関数が、その外部の変数を変更できるようにするために、渡された数値の実引数からアドレスを求めるための演算を行っているのだな。
  - 自分が今まで使ってきた C 言語よりも高級な言語においては、そのアドレスを求める処理を内部的に自動化しているものがあって、それを「**参照渡し**」というのだな。
  - 要するに C 言語では **全ての引数は値渡し** なのだな。
  - void ポインタってなんのためにあるんだ？
  - ポインタとは Windows における「ショートカット」機能である。
